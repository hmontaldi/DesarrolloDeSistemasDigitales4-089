Objetivo: Programar la interfaz y la funcionalidad del sistema que diseñaron en el TP N°2, siguiendo el prototipo aprobado. Duración estimada: 30 minutos.

🔗 Contexto

Este TP corresponde a la Etapa 4 — Desarrollo del ciclo de vida del software: acá escribimos el código del sistema según el diseño ya aprobado en el TP N°2. Recordá que esta es la etapa que menos defectos genera cuando el análisis de requisitos y el prototipo se hicieron bien.

📝 Consignas

1. Recuperar el diseño

20 pts

Antes de programar, revisá el prototipo y las pantallas que hiciste en el TP N°2.

  • Enumerá las pantallas o secciones que vas a construir.
  • Indicá qué datos va a mostrar u obtener cada una.

2. Estructura y estilo (HTML + CSS)

40 pts

Construí la interfaz del sistema respetando el diseño del prototipo.

  • Armá la estructura HTML de las pantallas definidas en la consigna 1.
  • Aplicá estilos con CSS (colores, tipografía, distribución) coherentes con lo prototipado.
  • El diseño debe verse ordenado y legible en pantalla.

3. Funcionalidad (JavaScript)

28 pts

Agregá la lógica necesaria para que el sistema haga lo que se definió como requisito funcional.

  • Programá al menos una funcionalidad interactiva (por ejemplo: validar un dato, mostrar/ocultar información, guardar o buscar algo).
  • Podés usar IA como apoyo para escribir o corregir código (ver nota más abajo).

4. Documentar el código

12 pts

Agregá comentarios breves en el código explicando qué hace cada parte importante.

🤖 Sobre el uso de IA

Está permitido usar herramientas de IA para generar o corregir código. La condición es que puedas explicar qué hace cada línea que entregás. Si en la puesta en común no podés explicar tu propio código, la consigna correspondiente no se considera resuelta.

✅ Criterios de evaluación

Consigna Se evalúa Puntos
1. Recuperar el diseño Coherencia con el prototipo del TP N°2 20
2. Estructura y estilo HTML válido, CSS aplicado, prolijidad visual 40
3. Funcionalidad El JavaScript funciona sin errores 28
4. Documentación Comentarios claros en el código 12
Entrega: archivo/s HTML, CSS y JS (o un único .html) subidos según indique el/la docente.
