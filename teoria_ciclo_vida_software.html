📋 Ciclo de Vida del Software

Teoría · Desarrollo de Sistemas Digitales · 5° Año · Prof. Humberto Montaldi · 2026

¿Qué es el Ciclo de Vida del Software? Es el conjunto de etapas ordenadas que todo equipo de desarrollo debe seguir para construir un sistema digital de calidad. No importa si el sistema es grande o pequeño: saltarse una etapa siempre genera problemas más adelante.

Pensalo como una receta: si te saltás un paso, el resultado final no sale bien.

1. Planificación

Se define el alcance del proyecto, los tiempos, el equipo y los recursos necesarios. Es la hoja de ruta del desarrollo.

2. Definición / Análisis de Requisitos 40-50% de defectos

Se identifica con detalle qué necesita el usuario y qué debe hacer el sistema. Es la etapa con mayor impacto: entre el 40% y 50% de los defectos del software se generan aquí si no se hace bien.

3. Diseño y Prototipo 15-20% de defectos

Se diseña la arquitectura del sistema, las pantallas y cómo fluyen los datos. Hacer prototipos permite mostrar la idea al usuario antes de programar y evitar correcciones costosas. Genera entre el 15% y 20% de los defectos si se descuida.

4. Desarrollo del Software

Los programadores escriben el código del sistema según el diseño aprobado. Para sorpresa de muchos, es la etapa que menos defectos genera cuando las etapas anteriores se hicieron bien.

5. Pruebas 10-15% de defectos

Se verifica que el sistema funcione correctamente y sin errores. Genera entre el 10% y 15% de defectos si no se le dedica tiempo suficiente. Una mala planificación hace que esta etapa sea siempre la que "paga las consecuencias" del retraso de las anteriores.

6. Lanzamiento / Despliegue

El sistema se instala y pone a disposición de los usuarios reales. Se capacita al personal y se monitorea el funcionamiento inicial.

7. Operaciones y Mantenimiento

El sistema está en uso. Se corrigen errores que aparecen, se agregan mejoras y se adapta a nuevas necesidades. Esta etapa puede durar años.

⚠️ Las 3 etapas que más defectos generan

Si estas tres etapas se hacen mal, pueden provocar entre el 65% y el 85% del total de los errores del producto final:

  • Etapa 2 — Análisis de Requisitos: el usuario no explica bien lo que necesita o el analista no pregunta suficiente. Resultado: el sistema hace algo distinto a lo esperado.
  • Etapa 3 — Diseño y Prototipo: se omite mostrar al usuario cómo va a quedar el sistema antes de programarlo. Resultado: hay que rehacer partes enteras.
  • Etapa 5 — Pruebas: se le dedica poco tiempo porque las etapas anteriores se atrasaron. Resultado: el sistema tiene errores que llegan al usuario final.

Conclusión: desarrollar buen software no es solo escribir buen código. Lo más importante es entender bien el problema y diseñar bien la solución antes de programar.

💡 Para pensar...

El 40% al 50% de los defectos de un sistema se generan en la etapa de Requisitos, pero los desarrolladores suelen pasar la mayor parte del tiempo en la etapa de Desarrollo. ¿Dónde habría que invertir más esfuerzo entonces?

📖 Glosario rápido

Requisito funcional
Lo que el sistema debe hacer (por ejemplo: "debe permitir buscar un libro por título").
Requisito no funcional
Cómo debe hacerlo (por ejemplo: "debe responder en menos de 2 segundos").
Prototipo
Versión simplificada y no funcional del sistema, usada para validar el diseño con el usuario antes de programar.
Defecto / Bug
Error en el software que provoca que no funcione como se esperaba.
Despliegue
Proceso de instalar y poner en producción el sistema para que los usuarios reales puedan usarlo.
Mantenimiento
Correcciones y mejoras que se realizan sobre el sistema una vez que está en uso.
Meta proceso
Proceso que organiza y coordina todos los demás procesos de un proyecto. El ciclo de vida es el meta proceso del desarrollo de software.
